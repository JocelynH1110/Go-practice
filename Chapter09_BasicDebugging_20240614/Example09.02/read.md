# 9-2 以套件做格式化輸出
## 9-2-1 fmt 套件
主要用途：將資料輸出到主控台（命令提示字元、Powershell、終端機）或傳回格式化後的字串。

## 9-2-2 fmt 的格式化輸出（fmt.Printf）
fmt.Printf() 可以使用更彈性方式組合不同的值、輸出想要的字串，也不需要先將那些值轉成字串。

```go
fmt.Printf("Hello %s, good morning", fname) 
```
以上例子：
* %s 稱為動詞（verb）、指定符號（specifier），這些會在格式字串中成為佔位符（placeholder）。
* fname 此值會放在格式字串中 %s 的地方。

### 格式化用的動詞
%b 印出 2 進制值
%d 印出 10 進位整數
%x 印出 16 進位值
%f 印出浮點數
%e 印出帶有科學記號的浮點數
%t 印出布林值
%s 印出字串

## 9-2-3 印出浮點數的進階格式化
* %.2f 表示印到小數後兩位（四捨五入）。
* %10.2f 表示分配 10 個字元的長度給浮點數，包括 1 個小數點、2 個數位，故整數部份的顯示長度便是 7 字元。
* 字數不足會把數字向右對齊、不足的部份空白填補。 

補零：
```go
fmt.Printf("%010.2f\n",1234.678915) //印出 0001234.68
```

靠左對齊：
```go
fmt.Printf("%-10.2f\n",1234.678915) //印出 1234.68
```

## 9-2-4 用 strconv.FormatFloat() 格式化浮點數
strconv 套件的功能：將其他型別的值轉換成字串。
但 FormatFloat() 在轉換過程中也可以指定要四捨五入的小數位數。


```
<格式化 string> := strconv.FormatFloat(<float64 浮點數> ,<格式化符號>, <小數位數>, <原始資料位元數>)
```
```go
strconv.FormatFloat(1234.678915,"f",2,64)   // 傳回 "1234.68"
```

* 傳入的浮點數為 float64 型別，得注意型別轉換。
* 格式化符號：跟 fmt 的格式化動詞相同，只是不用寫 % 。
* 最後一個參數：用來示意浮點數的原始值是 32 或 64 。

何時跟用 strconv 套件而不是 fmt ?
strconv 套件不須像 fmt 得判斷輸入的型別為何。
故處理效果更快且消耗記憶體更少。
當程式運作效率和記憶體用量是重要因素時，用 strconv 是更好的主意。
